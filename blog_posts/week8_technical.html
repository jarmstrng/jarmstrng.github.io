Active Record in Ruby/Rails Made Simple

Ruby/Rails's style of Active Record, object-relational mapping "Active record is an approach to accessing data in a database. A database table or view is wrapped into a class. Thus, an object instance is tied to a single row in the table." "The wrapper class implements accessor methods or properties for each column in the table or view." http://en.wikipedia.org/wiki/Active_record_pattern active record - object-oriented paradigm for relational databases- objects contain data and methods that can act on that data. There is a Class for the table, and an instance of that class is a row in the table. ActiveRecord - Ruby library - allows for inheritance of functionality from a parent class object-relational mapping "convert the object values into groups of simpler values for storage in the database (and convert them back upon retrieval)" http://en.wikipedia.org/wiki/Object-relational_mapping "Class methods are used to perform table-level operations, and instance methods perform operations on the individual rows." http://rubylearning.com/satishtalim/ruby_activerecord_and_mysql.html What it is Active record pattern is what you would call an object-oriented paradigm for relational databases. Objects contain data and methods that can act on that data, and so there is a Class for the table, and an instance of that class is a row in the table. The row/object has both data and some functions that can be used to act on that data. ActiveRecord is a Ruby library that uses the active record pattern. It comes with the Ruby on Rails framework but can also be used independently of Rails. It gives you handy methods to inherit and use so you can easily interact with your database in an object-oriented way. How it works "Class methods are used to perform table-level operations, and instance methods perform operations on the individual rows." http://rubylearning.com/satishtalim/ruby_activerecord_and_mysql.html Example Pros Cons Alternatives
